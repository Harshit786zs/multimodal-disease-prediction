# 🩺 Multimodal Disease Prediction System

A Streamlit web app that predicts the risk of **10 different diseases** using machine learning models trained on real medical datasets.

**🔗 Live demo:** https://multimodal-disease-prediction-btwkrnhvsuifphf6hdrlgu.streamlit.app

> ⚠️ This tool is for educational/demo purposes only and is **not** a substitute for professional medical diagnosis. Always consult a doctor for actual medical advice.

## Features

- Risk prediction across 10 diseases from a single app
- Per-prediction confidence score
- Session-based prediction history (sidebar)
- "Load example values" on every page for fast demoing
- Lightweight feature-importance view + optional SHAP per-prediction explanation

## Tech Stack

- **Frontend / app:** [Streamlit](https://streamlit.io)
- **ML:** scikit-learn, imbalanced-learn
- **Data handling:** pandas, numpy

## Project Structure

```
multimodal-disease-prediction/
├── app.py                  # Home page / entry point
├── utils.py                # Shared model loading, prediction, UI helpers
├── pages/                  # One file per disease (Streamlit multipage app)
├── outputs/                # Trained model_*.pkl / scaler_*.pkl files (+ optional *_encoders.pkl)
├── scripts/
│   └── export_encoders_example.py   # Template for saving real encoders during training
├── requirements.txt
└── README.md
```

> `notebooks/` or `src/training/` is not yet in the repo — see [Roadmap](#roadmap). Without it, this currently reads as inference-only; adding the training/EDA code is the single highest-value next step for credibility (academic or portfolio).

## Getting Started

```bash
git clone https://github.com/Harshit786zs/multimodal-disease-prediction.git
cd multimodal-disease-prediction
python -m venv venv
source venv/bin/activate   # venv\Scripts\activate on Windows
pip install -r requirements.txt
streamlit run app.py
```

The app expects trained model + scaler files in `outputs/`, named `<disease>_model.pkl` and `<disease>_scaler.pkl` (e.g. `diabetes_model.pkl`, `diabetes_scaler.pkl`).

## Known Limitations

- Several pages (Kidney, Stroke, Hypertension, Thyroid) re-derive categorical encodings by sorting option lists alphabetically to mimic `sklearn.LabelEncoder`. This only matches the real training-time encoding if the category set/order hasn't changed — see `scripts/export_encoders_example.py` for the fix (save the real fitted encoders during training).
- No automated tests yet.
- Datasets and training code aren't included in this repo.
